# CitePilot — 5-Minute Demo Script

Pre-flight (do BEFORE the audience is watching):

- `make up` + `make migrate` done; all four status pills green at `http://localhost:3000`.
- Seed project **"GraphRAG Survey — Related Work"** exists with the five papers imported (RAG/Lewis 2020, DPR, REALM, Atlas, Microsoft GraphRAG), graph enriched once, one successful compile.
- `main.tex` contains this paragraph (the selection target for step 2):
  > Retrieval-augmented generation grounds language model outputs in retrieved documents, but selecting which evidence to retrieve remains an open problem for scholarly writing.
- Run the whole flow once on the venue network so the Redis cache is warm. Keep one screenshot of a completed agent turn as the fallback if the LLM call fails live.

---

## 1. The hook (10 s)

Open the seeded workspace. One line: *"This is an Overleaf-style editor, except the AI assistant understands the citation graph — not just the text."*

## 2. Citation suggestions with reasons (90 s) — the core moment

Select the prepared paragraph → agent panel → "What related work should I cite here?" → Send.

Narrate while events stream: *"Watch the tool trace — it's running hybrid retrieval: vector similarity plus co-citation, bibliographic coupling, and shared concepts from Neo4j, fused with Reciprocal Rank Fusion."*

When the suggestion cards land, point at a reason: *"'Shares 12 references with relevant papers' — that reason is computed from graph features, never generated by the LLM. It cannot be hallucinated."*

## 3. Insert → BibTeX → PDF (60 s)

Click **Insert citation** on a card → `\cite{key}` lands in the draft → Save → **Compile** in the PDF panel → PDF renders. *"BibTeX came from Crossref publisher data, re-keyed to the project's stable citation key; compilation is sandboxed Tectonic with no network."*

## 4. The living graph (60 s)

Click a paper in the bibliography → citation graph renders (dashed amber = stubs). Hit **⟳ Enrich** → *"It's promoting the most-referenced stub papers to full papers right now — watch the bibliography densify."* Click a node to re-center.

## 5. Architecture, in words (60 s)

- *"Postgres is the source of truth; Neo4j is a derived mirror — I can wipe it and rebuild with one command."*
- *"Five retrieval signals stay independent until rank-only RRF fuses them — no scale-mixing between cosine similarity and citation counts."*
- *"The agent is a bounded tool loop: max 8 iterations, every tool call audited, errors fed back as data so the model self-corrects."*
- *"File edits are exact-anchor patches — a representation models are good at — with version checks and human approval in the UI."*

## 6. Technical encore, if the audience wants it (30 s)

```bash
npx @modelcontextprotocol/inspector docker compose exec -T backend python -m app.mcp_server.server
```

*"The same ten tools, exposed over MCP to Claude Desktop or any MCP client — capabilities are a layer; the web agent and the protocol are both just consumers."*

---

## If asked about limitations (answer honestly)

Single dev user (no auth yet) · one title+abstract chunk per paper (no full-text) · turn-level streaming, not token-level (the SSE protocol already supports upgrading) · MCP is stdio-only by design (HTTP exposure is a security boundary, not a config change).

## Live-failure fallbacks

- LLM call fails → show the prepared screenshot, keep narrating the tool trace story.
- OpenAlex slow → all demo queries are Redis-cached from pre-flight; re-run the same query.
- Compile fails → that IS a demo: show the failed status with the log excerpt, fix the typo, recompile.
